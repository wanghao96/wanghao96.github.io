设计模式_单一职责(Bridge、Decorator)

发表于 2019-06-23 | 更新于 2019-06-26

“单一职责”模式
在软件组件的设计中，如果责任划分的不清晰，使用继承得到的结果往往是随着需求的变化，子类急剧膨胀，同时充斥着重复代码，此时关键是划清责任。

桥接Bridge

桥接Bridge:将抽象部分与它的现实部分分离，使它们都可以独立地变化
解决问题：抽象和实现分离，便于扩展
实现：抽象桥接类内含抽象产品类指针

输出：
game computer
LOL running…
game computer
VS2017 running…

装饰Decorator

装饰Decorator:动态地给一个对象添加一些额外的职责，扩展功能方面，比子类方式更为灵活
解决问题：在不影响其他对象的情况下，以动态、透明的方式给单个对象添加职责；处理那些可以撤销的职责；当不能采用生成子类的方法进行扩充时。
实现：抽象装饰类继承抽象产品类时并依赖一个抽象产品类指针

输出：
踢腿
出拳
动作结束
